# 04 — Ansible (K3s + GitOps bootstrap)

**Goal:** one `ansible-playbook` run provisions K3s on all 3 nodes, installs ArgoCD via Helm,
creates the app namespace + secret, taints master, adds swap, and applies the root app — all
idempotent, all in the right order.
**Prereqts:** file 03 applied (`inventory.ini` exists); `kubernetes.core` + `community.general`
collections; an ansible-vault file holding the RabbitMQ credentials.
**Done when:** `kubectl get nodes` shows 3 Ready nodes from your laptop, ArgoCD is up on NodePort
30082 (HTTP) / 30083 (HTTPS), the `skywatch-rabbitmq` secret exists, and the root Application is applied. Re-running the
playbook changes nothing.

This file is where the **bootstrap ordering** (decided earlier) becomes structural instead of
"type the steps in the right order at 1 a.m."

---

## Repo layout

```
ansible/
  requirements.yml
  playbook.yml
  inventory.ini                  # generated by Terraform
  group_vars/all/vault.yml       # ansible-vault encrypted
  roles/
    k3s_master/tasks/main.yml
    k3s_worker/tasks/main.yml
    bootstrap/tasks/main.yml
```

## Step 1 — collections & vault

`requirements.yml`:
```yaml
collections:
  - name: kubernetes.core
  - name: community.general
```
`ansible-galaxy collection install -r requirements.yml`

Create the encrypted vault (never plaintext on disk, never in Git):
```bash
ansible-vault create group_vars/all/vault.yml
```
Contents:
```yaml
vault_rabbitmq_user: skywatch
vault_rabbitmq_password: <a-strong-password>
```

## Step 2 — `roles/k3s_master/tasks/main.yml`

Idempotency guard: only (re)install if k3s isn't already active.

```yaml
- name: Check if k3s is active
  ansible.builtin.command: systemctl is-active k3s
  register: k3s_status
  failed_when: false
  changed_when: false

- name: Install K3s server (pinned)
  ansible.builtin.shell: |
    curl -sfL https://get.k3s.io | \
      INSTALL_K3S_VERSION=v1.29.5+k3s1 \
      sh -s - server \
        --disable traefik \
        --tls-san {{ ansible_host }} \
        --node-taint node-role.kubernetes.io/control-plane=:NoSchedule
  when: k3s_status.stdout != "active"

- name: Wait for node-token
  ansible.builtin.wait_for:
    path: /var/lib/rancher/k3s/server/node-token

- name: Read join token
  ansible.builtin.slurp:
    src: /var/lib/rancher/k3s/server/node-token
  register: k3s_token_b64

- name: Set token fact
  ansible.builtin.set_fact:
    k3s_token: "{{ k3s_token_b64.content | b64decode | trim }}"

- name: Fetch kubeconfig to control machine
  ansible.builtin.fetch:
    src: /etc/rancher/k3s/k3s.yaml
    dest: "{{ playbook_dir }}/../kubeconfig"
    flat: true

- name: Rewrite kubeconfig server to public IP
  ansible.builtin.replace:
    path: "{{ playbook_dir }}/../kubeconfig"
    regexp: 'https://127.0.0.1:6443'
    replace: "https://{{ ansible_host }}:6443"
  delegate_to: localhost
```
> The master taint is applied at install time via `--node-taint` so no app pod ever lands there —
> no separate `kubectl taint` step needed.

## Step 3 — `roles/k3s_worker/tasks/main.yml`

Use the master **private** IP and pass the token as an **environment variable** to the installer
(the installer reads `K3S_TOKEN`/`K3S_URL` from env, not from CLI flags).

```yaml
- name: Check if k3s-agent is active
  ansible.builtin.command: systemctl is-active k3s-agent
  register: agent_status
  failed_when: false
  changed_when: false

- name: Join cluster as agent
  ansible.builtin.shell: |
    curl -sfL https://get.k3s.io | \
      INSTALL_K3S_VERSION=v1.29.5+k3s1 \
      K3S_URL=https://{{ master_private_ip }}:6443 \
      K3S_TOKEN={{ hostvars[groups['master'][0]]['k3s_token'] }} \
      sh -s - agent
  when: agent_status.stdout != "active"

- name: Label worker2 for app + monitoring placement
  ansible.builtin.command: >
    k3s kubectl label node skywatch-worker2 skywatch-role=app --overwrite
  delegate_to: "{{ groups['master'][0] }}"
  when: node_role == "worker2"
```
> Node hostnames come from `tags.Name` set in Terraform (`skywatch-master`, etc.). Confirm
> `kubectl get nodes` shows those names; if K3s used the EC2 internal hostname instead, set
> `--node-name skywatch-worker2` in the agent install.

## Step 4 — swap (worker2 only, for monitoring headroom)

In a small role or block, guarded by `creates:`:
```yaml
- name: Create 2G swapfile
  ansible.builtin.command: fallocate -l 2G /swapfile
  args: { creates: /swapfile }
- name: Set permissions
  ansible.builtin.file: { path: /swapfile, mode: '0600' }
- name: mkswap
  ansible.builtin.command: mkswap /swapfile
  register: mkswap
  failed_when: mkswap.rc != 0 and 'already' not in mkswap.stderr
  changed_when: "'Setting up swapspace' in mkswap.stdout"
- name: swapon
  ansible.builtin.shell: swapon /swapfile || true
  changed_when: false
```

## Step 5 — `roles/bootstrap/tasks/main.yml` (runs on localhost, talks to the cluster)

This is the ordered bootstrap. **Order matters and is enforced by task order**, not by you.

```yaml
# 0. point kubernetes.core at the fetched kubeconfig (set as env or module arg)

- name: Install ArgoCD via Helm
  kubernetes.core.helm:
    name: argocd
    chart_ref: argo/argo-cd            # add repo argo https://argoproj.github.io/argo-helm first
    chart_version: "7.*"               # ships ArgoCD v2.11.x
    release_namespace: argocd
    create_namespace: true
    values:
      global:
        nodeSelector: { kubernetes.io/hostname: skywatch-worker }
      server:
        service:
          type: NodePort
          nodePortHttp: 30082
          nodePortHttps: 30083
    wait: true

# GATE 1 — Application CRD must exist before we can apply a root Application
- name: Wait for Application CRD established
  kubernetes.core.k8s_info:
    api_version: apiextensions.k8s.io/v1
    kind: CustomResourceDefinition
    name: applications.argoproj.io
  register: appcrd
  until: appcrd.resources | length > 0
  retries: 30
  delay: 5

- name: Ensure skywatch namespace exists (BEFORE the secret)
  kubernetes.core.k8s:
    state: present
    definition:
      apiVersion: v1
      kind: Namespace
      metadata: { name: skywatch }

- name: Create the single rabbitmq secret (name contract!)
  kubernetes.core.k8s:
    state: present
    definition:
      apiVersion: v1
      kind: Secret
      metadata: { name: skywatch-rabbitmq, namespace: skywatch }
      type: Opaque
      stringData:
        RABBITMQ_DEFAULT_USER: "{{ vault_rabbitmq_user }}"
        RABBITMQ_DEFAULT_PASS: "{{ vault_rabbitmq_password }}"
        rabbitmq-username:     "{{ vault_rabbitmq_user }}"
        rabbitmq-password:     "{{ vault_rabbitmq_password }}"

- name: Apply the ArgoCD root application (app-of-apps)
  kubernetes.core.k8s:
    state: present
    src: "{{ playbook_dir }}/../argocd/root-app.yaml"
```

## Step 6 — `playbook.yml`

```yaml
- hosts: master
  become: true
  roles: [k3s_master]

- hosts: workers
  become: true
  roles: [k3s_worker]

- hosts: worker2_swap_target   # or target worker2 with a when: node_role == 'worker2'
  become: true
  tasks: []  # swap tasks here, guarded by node_role

- hosts: localhost
  connection: local
  gather_facts: false
  roles: [bootstrap]
```
> **Never run with `--limit`.** Limiting to workers skips the master play, so the `k3s_token` fact is
> never set and the join fails. Always run the whole playbook.

## Run

```bash
export KUBECONFIG=$(pwd)/../kubeconfig
ansible-playbook -i inventory.ini playbook.yml --ask-vault-pass
```

## Done when

- `kubectl get nodes` → 3 nodes Ready (`skywatch-master` tainted, no app pods on it).
- `kubectl get secret skywatch-rabbitmq -n skywatch` exists with the 4 keys.
- ArgoCD reachable: `http://<worker1-public-ip>:30082` (admin password from
  `kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath='{.data.password}' | base64 -d`).
- Re-run the whole playbook → every task reports **ok**, nothing **changed** (idempotent).
- ArgoCD will report the root app missing children until files 05–07 exist in Git — that's expected.
